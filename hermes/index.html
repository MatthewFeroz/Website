Hermes Gmail Automation

This page documents a private Google Workspace automation used by Matthew Feroz to manage his own Gmail inbox.

Last updated: May 31, 2026

App overview

Hermes Gmail Automation is a private, personal-use workflow operated by Matthew Feroz. It helps review, organize, label, archive, and manage messages in Matthew's own Gmail account. It is not a public consumer product, SaaS application, or service offered to outside users.

App purpose

The app is used to reduce inbox clutter by identifying promotional, spam-like, or non-relevant messages and helping move them out of the inbox while preserving access to messages in Gmail when appropriate.

Google user data accessed

Depending on the task Matthew authorizes, the app may request access to Gmail message metadata, labels, message content, threads, and send/modify capabilities. These permissions are used only to perform Matthew's requested Gmail management actions.

How Google user data is used

  • To search and review messages in Matthew's Gmail inbox.
  • To identify messages that appear promotional, spam-like, or unrelated to Matthew's current work.
  • To apply labels, mark messages, or archive messages when Matthew authorizes those actions.
  • To preserve important, personal, client, billing, event, and account-security messages.

Data storage

The app stores OAuth credentials needed to keep the automation connected to Matthew's Google account. Email content is not sold, rented, or used to build advertising profiles. Email content is accessed only as needed to complete the requested inbox-management workflow.

Data sharing

Google user data is not sold, rented, transferred, or shared with third parties for advertising, analytics, lead generation, or unrelated purposes. Data is used only to provide or improve this private automation for Matthew's own account.

Google API Services User Data Policy

This app's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

Security

Access is limited to Matthew's authorized environment. OAuth credentials are stored on Matthew's private infrastructure and are not intentionally exposed publicly. Access may be revoked at any time from the connected Google Account's security settings.

Data retention and deletion

OAuth credentials are retained only while Matthew wants the automation to remain connected. Matthew can revoke Google access at any time through his Google Account settings or by deleting the stored OAuth token from his infrastructure.

Terms of use

This app is provided for Matthew Feroz's private use. It is provided as-is without warranties of any kind. It should not be used by third parties or relied on as a public email-management service.
